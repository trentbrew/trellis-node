# Quick Start

The current SDK supports local mode and remote mode.

## Local mode

Local mode embeds the Trellis kernel directly and persists state to SQLite.

```ts
import { TrellisClient } from "trellis/client";

const db = new TrellisClient({ path: "./.trellis-db" });

const id = await db.create("Note", {
  title: "Local-first Trellis",
  pinned: "true",
});

const note = await db.read(id);
const result = await db.query('find Note where pinned = "true"');

console.log(note, result.bindings);

db.close();
```

## Remote mode

Remote mode talks to a running Trellis server over HTTP and uses WebSocket internally for query subscriptions.

```ts
import { TrellisClient } from "trellis/client";

const db = new TrellisClient({
  url: "http://localhost:3000",
  apiKey: "your-api-key",
});

const sub = db.subscribe('find Note where pinned = "true"', (rows, diff) => {
  console.log(rows, diff);
});

sub.unsubscribe();
db.disconnect();
```

Local-first runtime work will make local live subscriptions first-class without requiring the remote server path.

# Installation

Trellis is published as the `trellis` package and developed in the `trellis-package` codebase.

## Package usage

```bash
npm install trellis
```

Use package subpaths for specific surfaces:

```ts
import { TrellisClient } from "trellis/client";
import { startServer } from "trellis/server";
```

## Repository development

```bash
bun install
bun test
bun run build
```

## CLI usage

### Initialize with Progressive Disclosure

```bash
trellis init
```

The `trellis init` command now provides a two-stage onboarding experience:

**Stage 1: Setup Choice**
- **⚡ Minimal Setup**: Fast, one-shot setup with detected framework, zero plugins, and auto-detected IDE rules
- **🔧 Custom Guided Setup**: Customizes framework type, specific IDEs to target, workspace footprint sizes, and advanced feature plugins

**Stage 2: Success Screen with VCS + Semantic Substrate**

After initialization, you'll see both familiar VCS commands and the semantic substrate capabilities:

```bash
# VCS Commands (Stage 1)
trellis status          # Show repository state
trellis log             # View causal history
trellis issue list      # List issues and tasks
trellis milestone list  # Show narrative checkpoints

# Semantic Substrate (Stage 2 - revealed after init)
trellis web             # Launch local web client / graph visualizer
trellis query           # Run EQL-S semantic queries against your code graph
# Agent rules auto-configured for your IDE (Cursor/Windsurf)
```

### Daily Workflow Commands

```bash
trellis status
trellis log --limit 20
trellis issue list
trellis milestone list
```

## Docs app

The docs website lives under `apps/docs`.

```bash
npm --prefix apps/docs install
npm --prefix apps/docs run dev
```

The app is a Nuxt Content site. Canonical source docs live in `/docs`; published website content lives in `apps/docs/content`.

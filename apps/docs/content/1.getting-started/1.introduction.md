# Introduction

Trellis is a local-first graph for things that grow but can't hold their own weight: code, decisions, and the reasoning of AI agents. **Trellis Studio** is the workspace where you tend them.

```bash
npx trellis studio
```

For the engine-only path, Trellis is also a semantic graph where files, entities, facts, links, decisions, agents, branches, issues, and milestones live in one inspectable causal substrate.

The goal is not to build another hosted realtime database. The goal is to build an owned local graph runtime that can sync with peers, host agents, expose programmable views, and remain useful without a cloud server.

## What Trellis gives you

| Need | Trellis primitive |
|---|---|
| Local ownership | SQLite-backed kernel, op log, and blob store |
| Machine-readable state | Entity-Attribute-Value graph and EQL-S queries |
| Auditable change | Content-addressed causal operations |
| Human context | Issues, milestones, decisions, and Idea Garden clusters |
| Agent memory | Decision traces and graph-native tool records |
| Peer sync | Transport-independent op replication, with Iroh as the north-star transport |

## Architectural center

Trellis is designed so a useful node can run without a cloud server. Servers, relays, browser gateways, hosted indexes, and PartyKit-style rooms are compatibility adapters. They may bridge, accelerate, back up, or expose state, but they should not become the primary owner of state.

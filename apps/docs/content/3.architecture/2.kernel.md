# Kernel

The kernel is the source-of-truth layer for Trellis state. It owns graph mutation, operation recording, replay, persistence, and queryable EAV state.

## Core responsibilities

- Create, update, and delete entities
- Add and remove facts and links
- Append durable operations
- Replay operations into current state
- Persist snapshots and blobs
- Expose graph state to the query engine

## Current package locations

```text
src/core/kernel/
src/core/store/
src/core/query/
src/core/persist/
src/core/ontology/
```

## Design direction

Local mode should remain first-class. The kernel should not assume that a hosted server owns state. Remote server APIs are compatibility surfaces over the same local-first model.

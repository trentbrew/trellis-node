# Compatibility Adapters

Trellis should support the existing web ecosystem without becoming defined by it.

## Adapter types

| Adapter | Purpose |
|---|---|
| Trellis server | HTTP/WebSocket access to a Trellis node |
| Browser gateway | Let browser clients talk to local or remote Trellis state |
| PartyKit-style relay | Room fanout, browser-friendly realtime, presence, fallback routing |
| Hosted backup/index | Availability, search, public sharing, institutional workflows |
| Framework bindings | React, TSRX, Vue, Svelte, or other UI integrations |

## Rule

Adapters may bridge, accelerate, back up, or expose Trellis state. They should not become the primary owner of state.

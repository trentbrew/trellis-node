# Runtime Architecture

The runtime is the process boundary around the Trellis kernel. It should eventually support embedded, sidecar, local daemon, and NixOS service modes.

## Target shape

```text
Tauri UI
  -> local Turtle runtime
    -> Trellis kernel
      -> EAV graph
      -> causal op log
      -> query engine
      -> blob store
      -> identity and governance
    -> sync transports
    -> local API surfaces
```

## Responsibilities

The runtime should own:

- Kernel lifecycle
- Storage paths
- Identity and keys
- Local subscriptions
- Agent execution boundaries
- Sync transport lifecycle
- Browser gateway access when enabled

The UI should render and command the runtime, not own the runtime.

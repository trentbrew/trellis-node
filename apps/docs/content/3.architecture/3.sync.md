# Sync

Trellis sync should replicate graph operations between peers. The server is not the center of truth; it is one possible participant or relay.

## Transport-independent protocol

The sync protocol should support:

- Peer identity
- Have/want/ops/ack style exchange or equivalent
- Idempotent operation application
- Causal validation
- Blob transfer
- Reconnect and catch-up
- Conflict and capability policy

## Iroh direction

Iroh is the preferred long-term peer-to-peer transport for local-first Trellis nodes. It fits Trellis because the graph is already op-log-oriented and content-addressed.

## Relay direction

Relays remain useful for browsers, unreachable peers, backups, public sharing, and hosted indexes. They should accelerate or bridge sync, not own primary state.

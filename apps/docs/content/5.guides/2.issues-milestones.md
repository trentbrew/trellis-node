# Issues and Milestones

Issues and milestones serve different roles.

## Issues are plans

```bash
trellis issue create -t "Prototype local Trellis runtime loop" -P high \
  -l runtime,local-first,sdk \
  --ac "Demo creates, updates, and queries graph entities without a remote server"
```

Use issues for planned work, acceptance criteria, labels, and status.

## Milestones are narrative history

```bash
trellis milestone create -m "Move Trellis docs into canonical package codebase"
```

Use milestones after a coherent checkpoint is completed.

# TrellisVCS Workflow

TrellisVCS is graph-native version control. It is not Git.

## Start work

```bash
trellis status
trellis log --limit 20
trellis garden list
```

Check the Idea Garden before starting new work so abandoned context can be reused.

## Work normally

When `trellis watch` is running, file changes are recorded as immutable operations. There is no staging area.

## Create narrative checkpoints

```bash
trellis milestone create -m "Implement local runtime loop"
```

Milestones summarize coherent completed work. Use issues for plans and acceptance criteria.

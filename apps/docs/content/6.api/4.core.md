# Core APIs

The `trellis` package exposes modular subpaths.

| Subpath | Purpose |
|---|---|
| `trellis/core` | EAV graph, kernel, query, ontology, agents, plugins |
| `trellis/client` | Local and remote SDK |
| `trellis/server` | HTTP, WebSocket, auth, permissions, tenancy |
| `trellis/react` | React provider and hooks |
| `trellis/vcs` | Causal stream, branches, milestones, merge, issues |
| `trellis/decisions` | Decision trace capture and query helpers |
| `trellis/links` | Wiki-link parsing and reference resolution |
| `trellis/ai` | Embedding and semantic search surfaces |

Use the subpath that matches the layer you are integrating with.

# Protocol Overview

Trellis is also a protocol for machine-readable knowledge lifecycles: how entities, operations, trajectories, ontologies, and provenance can be represented, exchanged, replayed, and queried.

## Protocol layers

| Layer | Role |
|---|---|
| EAV graph | Flexible entity, fact, and link representation |
| Causal ops | Durable event-sourced mutation history |
| EQL-S | Structured query language over graph state |
| JSON-LD | Linked Data-compatible serialization target |
| PROV-O alignment | Provenance vocabulary for activities, agents, and artifacts |
| Trajectories | Replayable agent or workflow sessions |
| Ontologies | Domain-specific schemas and capabilities |

## Public-good direction

The spec, ontologies, trajectory format, and reference corpus should be open and portable. Implementations can compete downstream, but the substrate should remain a commons.

# Trajectories

A trajectory is a replayable record of a knowledge workflow or agent session.

In the Trellis protocol direction, trajectories should be serializable as machine-readable artifacts that link:

- Inputs
- Tool calls
- Model calls
- Human decisions
- Generated artifacts
- Graph mutations
- Evaluation results
- Provenance metadata

## Why trajectories matter

Knowledge work is increasingly AI-assisted, but outputs alone are insufficient. A paper, code change, or dataset should be inspectable through the path that produced it.

## Replay direction

A replay tool should be able to take a trajectory, run it against a supported model or runtime, diff outputs, and produce a comparison report.

# Local-First OS Runtime

Trellis is the local-first graph substrate beneath Turtle: an OS-like runtime where personal and collaborative knowledge is owned locally, queryable by machines, and syncable with peers.

## The invariant

A fully useful Trellis node must run without a cloud server.

This means local state, local identity, local persistence, local query execution, and local agent memory are primary. Cloud services are optional distribution and compatibility layers.

## Target stack

```text
NixOS / host system
  -> Turtle runtime service or sidecar
    -> Trellis kernel
    -> local SQLite/op-log/blob store
    -> local query engine
    -> local agent memory
    -> Iroh peer sync
  -> Tauri desktop shell
  -> optional browser gateway or relay
```

## Why not cloud-first

Cloud-first systems make the server the owner of truth and treat local state as cache. Trellis reverses this. The local graph is truth; hosted servers may relay, index, back up, or expose state, but they should not become the authority.
